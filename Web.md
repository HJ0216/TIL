### Cookie와 Session

- Cookie

  - 브라우저가 관리하는 클라이언트 측 저장소
  - 서버가 Set-Cookie 헤더로 쿠키를 발급

    `Set-Cookie: theme=dark; Expires=Sat, 01 Jan 2026 00:00:00 GMT; Path=/`

  - 브라우저는 쿠키를 저장하고, 이후 해당 도메인으로 요청할 때 자동으로 Cookie 헤더에 담아 전송

    `Cookie: theme=dark`

- Session
  - 세션은 서버가 관리하는 사용자 상태 저장소
  - 서버는 사용자가 로그인 등 특정 요청을 하면 세션 객체를 생성하고,
    이를 서버 메모리 또는 Redis/DB 등에 저장
  - 생성된 세션에 고유한 Session ID를 부여하고, 이 Session ID만을 브라우저에 쿠키로 저장
  - 이후 요청 시 브라우저는 Session ID를 함께 전송하여 서버가 해당 사용자 정보를 식별

| 항목            | 세션(Session)                   | 쿠키(Cookie)                          |
| --------------- | ------------------------------- | ------------------------------------- |
| **저장 위치**   | 서버                            | 클라이언트(브라우저)                  |
| **보안**        | 비교적 안전                     | 변조 위험 있음                        |
| **유지 기간**   | 브라우저 종료 시 만료(기본)     | 만료일 지정 가능                      |
| **적합한 용도** | 로그인, 장바구니 등 민감한 정보 | 사용자 설정, UI 환경 등 비민감 데이터 |
